# Support Tickets API Documentation

This document describes the OVHcloud Support API endpoints available for managing support tickets in the `n8n-nodes-ovhcloud` package. These endpoints allow you to list, create, close, reopen, and reply to support tickets.

## Overview

The Support API provides a set of RESTful endpoints for interacting with support tickets through the OVHcloud API. It is part of the OVHcloud Public Cloud API and requires appropriate IAM permissions for authentication.

---

## API Endpoints

### Base URL

`https://eu.api.ovh.com/v1`

### Endpoints Summary

| Path                                      | HTTP Method | Description                                       |
| ----------------------------------------- | ----------- | ------------------------------------------------- |
| `/support/tickets`                        | GET         | List support tickets identifiers for this service |
| `/support/tickets/create`                 | POST        | Create a new ticket                               |
| `/support/tickets/{ticketId}`             | GET         | Get a specific ticket                             |
| `/support/tickets/{ticketId}/canBeScored` | GET         | Check whether a ticket can be scored              |
| `/support/tickets/{ticketId}/close`       | POST        | Close a ticket                                    |
| `/support/tickets/{ticketId}/messages`    | GET         | Get ticket messages                               |
| `/support/tickets/{ticketId}/reopen`      | POST        | Reopen a ticket                                   |
| `/support/tickets/{ticketId}/reply`       | POST        | Reply to a ticket                                 |
| `/support/tickets/{ticketId}/score`       | POST        | Set ticket score                                  |

---

## Authentication

All Support API endpoints require authentication via the OVHcloud API. You must provide valid credentials in your requests.

### Required IAM Actions

- `account:apiovh:support/tickets/get`
- `account:apiovh:support/tickets/create`
- `account:apiovh:support/tickets/close`
- `account:apiovh:support/tickets/reopen`

---

## Parameters

### Common Parameters

#### Query Parameters

- **archived** (boolean): Search archived tickets. Optional.
- **category** (support.TicketCategoryEnum): Search by ticket category. Optional.
- **customerReplyNeeded** (boolean): Tickets pending a customer reply. Optional.
- **excludeAutogenerated** (boolean): Exclude auto-generated tickets. Optional.
- **maxCreationDate** (datetime): Maximum creation date. Optional.
- **minCreationDate** (datetime): Minimum creation date. Optional.
- **page** (long): Page of tickets to fetch (only if pageSize is defined). Optional.
- **pageSize** (long): Number of tickets to fetch. Optional.
- **product** (support.TicketProductEnum): Search by ticket product. Optional.
- **serviceName** (string): Ticket message service name. Optional.
- **status** (support.TicketStatusEnum): Status of ticket. Optional.
- **subject** (string): Search by ticket subject. Optional.
- **ticketNumber** (string): Search by ticket number. Optional.

#### Body Parameters

- **body** (text): Ticket message body. Required for creating and replying to tickets.
- **category** (support.TicketCategoryEnum): Ticket message category. Optional.
- **impact** (support.TicketImpactUrgencyEnum): Ticket impact (reserved for Business or Enterprise support level accounts). Optional.
- **product** (support.TicketProductEnum): Ticket message product. Optional.
- **serviceName** (string): Ticket message service name. Optional.
- **subcategory** (support.TicketSubCategoryEnum): Ticket message subcategory. Optional.
- **subject** (string): Ticket message subject. Required for creating a new ticket.
- **type** (support.TicketTypeEnum): DEPRECATED - Ticket type (criticalIntervention requires VIP support level). Optional.
- **urgency** (support.TicketImpactUrgencyEnum): Ticket urgency (reserved for Business or Enterprise support level accounts). Optional.
- **watchers** (string[]): List of e-mail addresses to notify on ticket updates (max. 10). Optional.

## Usage Examples

### Example 1: List Open Tickets

```typescript
// List all open tickets for a specific service
const openTickets = await this.ovhCloudApiClient.httpGet('/support/tickets', {
    serviceName: 'myServiceName',
    status: 'open',
});
```

### Example 2: Create a New Ticket

```typescript
// Create a new ticket with subject and body
const newTicket = await this.ovhCloudApiClient.httpPost('/support/tickets/create', {
    subject: 'My ticket subject',
    body: 'This is the message body of my ticket',
    category: 'incident',
});
```

### Example 3: Get Ticket Details

```typescript
// Retrieve details for a specific ticket
const ticketDetails = await this.ovhCloudApiClient.httpGet('/support/tickets/{ticketId}', {
    ticketId: 12345,
});
```

### Example 4: Reply to a Ticket

```typescript
// Reply to an existing ticket
const replyResponse = await this.ovhCloudApiClient.httpPost('/support/tickets/{ticketId}/reply', {
    ticketId: 12345,
    body: 'This is my reply message',
});
```

### Example 5: Close a Ticket

```typescript
// Close a ticket
const closeResponse = await this.ovhCloudApiClient.httpPost('/support/tickets/{ticketId}/close', {
    ticketId: 12345,
});
```

### Example 6: Reopen a Ticket

```typescript
// Reopen a closed ticket with a reason
const reopenResponse = await this.ovhCloudApiClient.httpPost('/support/tickets/{ticketId}/reopen', {
    ticketId: 12345,
    body: 'Reason for reopening this ticket',
});
```

### Example 7: Set Ticket Score

```typescript
// Score a ticket (if scoring is enabled)
const scoreResponse = await this.ovhCloudApiClient.httpPost('/support/tickets/{ticketId}/score', {
    ticketId: 12345,
    score: 'good',
    scoreComment: 'The support was very helpful',
});
```

## Error Handling

All endpoints may return errors in the following cases:

- **Invalid authentication**: If credentials are missing or incorrect
- **Missing required parameters**: For operations requiring specific parameters
- **Invalid parameter values**: If provided parameters don't match expected types or values
- **IAM permission errors**: If the authenticated account lacks required permissions
- **Rate limiting**: If too many requests are made in a short period

### Common Error Responses

- **401 Unauthorized**: Invalid or missing authentication credentials
- **403 Forbidden**: Missing required IAM permissions
- **404 Not Found**: Invalid ticket ID or resource path
- **400 Bad Request**: Invalid parameter format or missing required fields

## Rate Limits & Quotas

- Maximum of 10 email addresses can be specified in the `watchers` parameter
- All endpoints follow OVHcloud API rate limiting policies
- Refer to [OVHcloud API documentation](https://api.ovh.com/) for specific rate limits

---

## Best Practices

- Use appropriate **categories** and **subcategories** when creating tickets to ensure proper routing
- Provide clear **subject** lines to help support teams quickly understand the issue
- Include detailed **body** information with specific error messages or descriptions
- Use **watchers** to notify relevant team members about updates
- Score tickets only when appropriate and provide meaningful **scoreComment**

---

## Support Levels

Some parameters like `impact`, `urgency`, and `watchers` are reserved for specific support levels:

- **Business or Enterprise**: Can use `impact`, `urgency`, and `watchers` parameters
- **Standard**: Limited to basic ticket operations without scoring capabilities

---

## Notes

- The `type` parameter is deprecated and should not be used
- All dates should be provided in ISO 8601 format (YYYY-MM-DDTHH:MM:SSZ)
- Ticket identifiers are internal and may not match external ticket numbers
- Refer to the [OVHcloud Support documentation](https://docs.ovh.com/) for additional details
